# Discussions Guide

GitHub Discussions are for conversation. Issues are for reproducible work.

Use Discussions when you want to:

- show a tuned preset;
- ask how to integrate the renderer into another project;
- compare shader behavior;
- propose an optical direction before it becomes a concrete issue;
- share screenshots, recordings, or presets.

Use Issues when you have:

- a reproducible bug;
- broken build or test output;
- a specific feature request with acceptance criteria.

Issue triage rules live in [Issue Triage](issue-triage.md).

## Recommended Categories

Existing GitHub categories:

- `Announcements` for release notes and maintainer updates.
- `General` for broad project questions.
- `Ideas` for early feature proposals.
- `Q&A` for integration help.
- `Show and tell` for screenshots, clips, presets, and tuned outputs.

Recommended manual additions:

- `Shader tuning` for optical presets, parameter questions, and refraction behavior.
- `Integration help` for moving the effect into another app, canvas, DOM, or WebGL source.

## Category Forms

Discussion form templates live in:

```text
.github/DISCUSSION_TEMPLATE/
```

They ask for source type, preset JSON, browser/GPU context, screenshots, and what the user is trying to improve.
